# Отчёт: бот CCGram снова молчал — доведено до конца

Дата: 2026-07-06. Работа только над ботом (`~/.ccgram/`), приложение hours-calculator не трогалось.
(Прошлый отчёт — про N+1 в выплатах — в git-истории.)

## 1. Задеплоился ли прошлый фикс (разбивка 4096) в живого бота?

**ДА.** Проверил: рабочий процесс бота (pid стартовал в 11:57, после правки файла в 11:57),
в живом файле есть `splitForTelegram`/`_sendChunked`. После моего перезапуска основной бот
**не логировал** ни одной новой ошибки «text is too long». То есть прошлый фикс работает.

**НО молчание вернулось по ДРУГОЙ причине** — не из-за основного бота.

## 2. Настоящая причина 30-минутного молчания

Уведомление о **завершении** задачи присылает **не основной бот**, а **отдельный процесс** —
хук `enhanced-hook-notify.js`, который Claude Code запускает по событию Stop (задача закончена).
Мой прошлый фикс был в основном боте и этого хука не касался. У хука своя логика, и там было
**подавление уведомления**:

- Хук считает задачу «телеграмной» по файлу-маркеру `typing-active`. Основной бот **удаляет
  этот маркер через 5 минут**. Твоя задача шла ~30 минут → к моменту завершения маркер уже исчез.
- Тогда включается проверка «пользователь активен за терминалом» — она читает **глобальный**
  файл `/tmp/claude_last_msg_time`, общий для **всех** сессий Claude. Его «освежает» любая
  активность (другая сессия за Mac, сам бот и т.д.). Из-за этого проверка считала тебя
  «активным» → уведомление **молча подавлялось** (`return`), даже без записи в лог (хук писал
  ошибки в `console`, а не в файл — поэтому в логах ничего и не было).

Итог: задача выполнилась и закоммитилась, а «готово» было **подавлено** и **невидимо**.

## 3. Что починил (файлы в `~/.ccgram/dist`)

1. **`enhanced-hook-notify.js` → `completed` и `waiting` теперь `alwaysNotify: true`.**
   Уведомление о завершении и об ожидании ввода **приходит всегда**, без хрупкой проверки
   «активен за терминалом». Это главный фикс молчания.
2. **`enhanced-hook-notify.js` → разбивка длинных сообщений (≤4000)** + при провале Markdown →
   простой текст + **явная запись ошибки в файл** `~/.ccgram/logs/hook-notify.log` (раньше
   провал был невидим — теперь молчание оставит след).
3. **`workspace-telegram-bot.js` → маркер `typing-active` живёт 5 мин → 3 часа.** Длинная
   телеграм-задача теперь остаётся «телеграмной» весь срок → и завершение, и **запросы
   разрешений** (`permission-hook.js`) корректно уходят на телефон, а не подавляются. Обычные
   сессии за Mac не затронуты (у них маркера нет).
4. Прошлый фикс разбивки в основном боте — на месте, не откатывал.
5. Бэкапы оригиналов: `*.bak-before-fixes-20260706` рядом с файлами.

`permission-hook.js` **намеренно не трогал** его логику (иначе твои разрешения при работе
прямо за Mac уходили бы на телефон и блокировали десктоп-сессии) — вместо этого продлил
маркер (п.3), чего достаточно.

## 4. Проверил ли РЕАЛЬНОЙ отправкой? — ДА

Запустил хук завершения **ровно как это делает Claude Code**:
`echo '<payload>' | node dist/enhanced-hook-notify.js completed` — дважды. Оба раза:
**exit 0**, файл `hook-notify.log` пуст (отправка не падала), токен/чат реально подгружены
из `.env` (проверено: `TELEGRAM_ENABLED=true`, токен длиной 46, chat `7180071033`). Плюс
контрольное сообщение через API — **доставлено (ok:true)**. Тебе в Telegram пришли тестовые
«✅ Task completed in hours-calculator …».

## 5. Работает ли уведомление о завершении?

**Да** — теперь оно `alwaysNotify` и приходит гарантированно. Разбивка и лог ошибок
закрывают остальные сценарии тишины (длинный ответ, кривой Markdown, сбой отправки).

## 6. Важный нюанс / что учесть

- **Побочный эффект `alwaysNotify: true`:** уведомление о завершении теперь приходит **для
  любой** сессии Claude на Mac, включая работу прямо за компьютером (не только телеграм-задачи).
  Это осознанный размен: лучше «лишнее» уведомление, чем пропущенное (ты просил надёжность).
  Если станет шумно — скажи, переключу на «только телеграм-задачи» (через маркер, без alwaysNotify).
- ⚠️ Все правки — в **скомпилированном** `dist`. **Обновление/переустановка CCGram затрёт**
  и эти фиксы, и прошлые, и переводы. Тогда напиши — верну.
- Служба `com.ccgram` (launchd, 24/7) перезапущена, бежит новый код (pid новый, старт чистый).
