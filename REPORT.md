# Отчёт: аудит и доработка Telegram-бота CCGram

Дата: 2026-07-06. Приложение hours-calculator **не трогалось** — работа только над ботом
(`~/.ccgram/`) и правилом языка в `CLAUDE.md`. (Прошлый отчёт — про закрытие /docs — в git-истории.)

## 1. Что нашёл — КОРНЕВАЯ ПРИЧИНА

**Задачи не терялись — бот их получал, но не мог прислать ответ.**

В логах (`~/.ccgram/logs/bot-stderr.log`) висела ошибка:
`Telegram API error: Bad Request: text is too long`.

Причина: функция `sendMessage` слала текст в Telegram **без разбивки**, а у Telegram
жёсткий лимит — **4096 символов** на сообщение. Как только ответ Claude или вывод
`/status` превышал лимит — отправка падала, сообщение не доходило, бот «молчал». Поэтому:
- **`/status` не отвечал** — вывод сессии почти всегда длиннее 4096.
- **«Задачи терялись»** — промпт доходил до бота (видно в логах), но длинный ответ
  на него не мог быть доставлен.

Второй, менее заметный источник тишины: «кривая» разметка Markdown в тексте Claude
(несбалансированные `*`/`_`) — Telegram тоже отклонял сообщение целиком.

## 3. Что починил

- **Разбивка длинных сообщений** (`sendMessage`/`sendHtmlMessage` в
  `~/.ccgram/dist/workspace-telegram-bot.js`): текст режется на куски ≤4000 символов и
  шлётся частями; если Markdown не парсится — повтор простым текстом (ещё один источник
  тишины закрыт). Юнит-тест: 21000 символов → 6 корректных кусков; короткое → 1 кусок.
- Бэкап оригинала: `~/.ccgram/dist/workspace-telegram-bot.js.bak-before-fixes-20260706`.

## 4. Кнопочный UX (Часть 2) — ДА, сделано

Новый режим: **отправляешь боту обычный текст задачи → появляется карточка с кнопками:**
- **📁 Проект** — активные сессии (🟢) и недавние проекты
- **🧠 Модель** — Haiku (дёшево) / Sonnet (обычно) / Opus (сложное)
- **⚡ Усердие** — Низкое / Среднее / Высокое
- **▶️ Запустить**

Выбранное отмечается ✅. **Последний выбор запоминается** (`~/.ccgram/src/data/last-choices.json`)
и предлагается по умолчанию — для похожей задачи можно просто жать «Запустить». По
«Запустить» бот применяет модель и усердие и отправляет твой текст в выбранный проект.
Старые команды работают как раньше. Если карточка падает — безопасный откат, задача не теряется.

**Важное ограничение (честно):** в CCGram «проект» = **уже запущенная сессия Claude**.
Поэтому чтобы кнопки заработали, нужна хотя бы одна активная сессия — один раз сделать
`/new hours-calculator`. Запуск новой сессии «с нуля» одним тапом не делал — это гонка
по времени, рискованно без живого теста. Встроенная альтернатива: если сессии нет,
«Запустить» стартует проект и просит отправить задачу ещё раз.

и перезагрузки Mac. Сейчас `state = running`. Рестарт: `launchctl kickstart -k gui/$(id -u)/com.ccgram`.
